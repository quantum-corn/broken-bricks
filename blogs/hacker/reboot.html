The Name Of The Spell

In the early 1990's, a student at University of Helsinki, named Linus Torvalds took up a personal project - to create a free operating system, or OS, similar to MINIX, which was a UNIX like OS.

Here, a note on the history of OS is in order. The very first computers were huge devices that did relatively simple mathematical calculations, for these, an OS was not needed. The next generation of computers were relatively more capable in the sense that they could process a batch of data and this called for a software between the machine and the actual program doing the processing to manage the batch of tasks to be done, this was the first ever OS. Then come in, multiprocessing capable computers, and naturally a need for more capable OS, now to manage the batch of tasks to be done as well as the various tasks being done simultaneously. This is where OS started being taken seriously and names like UNIX came up. To keep it simple UNIX was pretty much one of the very first multitasking capable OS and even more so one that is still alive 33 years later. The UNIX was licensed to many organizations which modified the original UNIX to suit their needs better. Soon some independent creators made OS similar to UNIX. One such was MINIX, as in mini-UNIX. Then we get personal computers in the last quarter of 20th century and then it was increasingly important that OS be accessible to as many users as possible, as not everyone using a personal computer was expected to a professional. And that is when Microsoft rolls out as a budding software firm, which Apple hires to make the OS for it's PC's. But as Apple rolls out Mac OS, Microsoft decides to roll out it's own OS - Windows, and then comes the revolution in personal computing, the 'Graphical User Interface or GUI'.

An OS is basically just an agent that does the talking between the application, which is the program that determines what needs to be done and the machine that does it. In other words, the application tells the OS, what needs to be done, and the OS collects this list of 'what' from various applications, or various 'what' from the same application and tells it to the machine, one at a time in an order and manner that best utilizes the resources available, so as to complete all the 'what', that needed to be done in the most efficient manner, and takes the result of the process and gives it back to the application that asked for it. But that would be the minimal OS, or the kernel, as we call it. There is a lot of room there to add in various utilities, that enhance various functions. Like GUI, the real time digital clock we are so used to having on our computers, the access to internet, the ability to play multimedia files are all utilities. So, as things stand, an actual acceptable OS has a kernel and a set of utilities.

The Age Of Origin

Now let us get back to Linus. In 1994, comes the v1.0 of the kernel of this operating system, which he called Linux, from LINUs and uniX. But this was just the kernel. Around the same time FSF, or Free Software Foundation, set out to make their own free OS, which they called GNU, which comic as I find it, stands for GNU's Not UNIX, (yes it is a recursive acronym, i.e. the GNU in the expanded form has the same expansion again, infinitely) but they started with working on the utilities first. And as it happened, they decided to use Linux kernel and thus we had out first complete GNU/Linux, or what we commonly call Linux. And it was available for free for everyone, and continues to be so, to this day. Now various hobbyists and professionals got their hands on this. And what do you do when you get a new house? You modify it to suit your needs and tastes. So these people did the same. And thus we have various modifications on Linux, and that's what distros are. An undeniably important thing to mention here would be that since it was free, anyone could work on it, and that was a key reason it developed much more quickly, variedly and yet flexibly, that today the majority of the internet runs on servers running Linux, Android is based on Linux, and even supercomputers run Linux variants.

And with that, I will see you around.
